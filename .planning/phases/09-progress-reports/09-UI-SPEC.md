| Token | Value | Usage in this phase |
|-------|-------|---------------------|

| sm | 8px | Stack gaps between check-in field label and input; chip row gap |
| md | 16px | Card internal horizontal padding; check-in field vertical spacing; tab bar gap |
| lg | 24px | Section-to-section vertical spacing; page top padding |
| xl | 32px | Bottom padding above bottom nav (combined with `pb-28` pattern = 112px) |
| 2xl | 48px | Not used directly — absorbed into `pb-28` bottom nav clearance |

- Bottom nav clearance: `pb-28` (112px) — matches established pattern from `NutritionDashboard.tsx` (`px-5 pt-6 pb-28`)

- Sparkline SVG: `width=120 height=40` — fixed dimensions for inline SVG within card; not a spacing token

## Typography

All sizes are pre-populated from the existing codebase. No new type styles are introduced; Phase 9 uses the same 4-level hierarchy established in Phases 2–8.

| Role | Size | Weight | Line Height | Usage in this phase |
|------|------|--------|-------------|---------------------|
| Body | 14px (`text-sm`) | 400 regular | 1.6 (`leading-relaxed`) | Check-in field descriptions; weekly report body copy; behavior win chip labels; focus suggestion paragraph |
| Label | 12px (`text-xs`) | 700 bold (`font-bold`) | 1.4 | Section sub-headings ("پیروی این هفته"), metric unit suffixes (kg, ساعت), tab labels, empty-state supporting text |
| Heading | 20px (`text-xl`) | 700 bold (`font-bold`) | 1.3 | Weekly report section heading; progress page title |
| Display | 24px (`text-2xl`) | 700 bold (`font-bold`) | 1.2 | Latest weight number displayed prominently; greeting line on progress page |

**Font weight rule:** Only 2 weights are used: 400 (regular) and 700 (bold). Never use 300, 500, 600, or 800.

**Numeric display note:** Weight values (e.g. `72.5` kg) display at 24px bold. Delta values (e.g. `−1.2 kg`) display at 14px with color coding (`text-success` for loss toward goal, `text-warm` for gain, `text-ink-2` for neutral).

## Component Inventory

New components for this phase only. All inherit the existing visual pattern (rounded-2xl cards, shadow-sm, bg-elevated, px-5).

### 1. `CheckInForm.tsx`
- Full-width card (`rounded-2xl bg-elevated p-5 shadow-sm`)
- Fields: weight (number input), hunger 1–5 (segmented tap row), sleep (number input), stress 1–5 (segmented tap row), activity (number input), adherence notes (textarea)
- Each field: `<label>` at 12px bold + input at 14px regular
- Input style: `w-full px-4 py-3 rounded-2xl bg-surface border border-line text-sm text-ink placeholder:text-ink-3 focus:outline-none focus:border-brand transition-colors` (matches `MealAnalysisForm.tsx` exactly)
- 1–5 scale selectors: horizontal row of 5 buttons, each `w-10 h-11 rounded-xl text-sm font-bold border transition-colors`; selected state: `bg-brand text-elevated border-brand`; unselected: `bg-surface border-line text-ink-2`
- Submit button: `w-full py-4 rounded-2xl bg-brand text-elevated font-bold text-sm disabled:opacity-60 transition-opacity` (matches established CTA pattern)
- Success feedback: inline green confirmation text (`text-success text-sm font-bold`) replaces error slot — no modal, no toast

### 2. `WeightSparkline.tsx`
- Inline SVG only — zero external dependencies
- Props: `weights: number[], width?: number (default 120), height?: number (default 40)`
- Returns `null` when `weights.length < 2`
- Path: `stroke="var(--color-brand)"`, `strokeWidth="2"`, `strokeLinecap="round"`, `strokeLinejoin="round"`, `fill="none"`
- Guard: `const range = max - min || 1` prevents divide-by-zero on flat data
- Accessible: `aria-hidden="true"` on the SVG element; weight delta text is the accessible label

### 3. `ProgressSummary.tsx`
- Outer scroll container: `px-5 pt-6 pb-28 space-y-6` (matches NutritionDashboard)
- Weight card: `rounded-2xl bg-elevated p-5 shadow-sm` containing latest weight (24px bold) + `WeightSparkline` side-by-side + delta indicator
- Behavior wins section: heading at 14px bold + horizontal `flex-wrap gap-2` row of achievement chips
- Achievement chip: `px-3 py-1.5 rounded-full text-xs font-bold` — achieved: `bg-brand-muted text-brand`; not achieved: `bg-surface border border-line text-ink-3`
- Logging streak: small badge inline with a `ClipboardList` icon (16px) + "X روز متوالی" text at 12px

### 4. `WeeklyReport.tsx`
- Card container: `rounded-2xl bg-elevated p-5 shadow-sm space-y-4`
- Metric rows: two-column layout (`flex items-center justify-between`) — label at 12px `text-ink-3` + value at 14px `text-ink font-bold`
- Sleep/stress note: amber tint banner (`rounded-xl bg-warm-muted p-4 text-sm text-ink-2`) — only rendered when `sleep_stress_note` is non-null
- Focus suggestion card: `rounded-xl bg-brand-muted p-4` — heading "پیشنهاد هفته بعد" at 12px bold `text-brand` + body at 14px `text-ink-2 leading-relaxed`
- Adherence percentage: `text-2xl font-bold text-brand` displayed prominently inside a centered metric cell

### 5. Tab Bar (inline in `ProgressScreen.tsx`)
- Two tabs: "خلاصه" (summary) and "گزارش هفتگی" (weekly)
- Tab bar style: `flex border-b border-line mb-5`
- Each tab: `flex-1 py-3 text-sm font-bold text-center transition-colors`; active: `text-brand border-b-2 border-brand`; inactive: `text-ink-3`
- Not a router-level tab — client-side state toggle (`useState<'summary' | 'weekly'>`)

## Interaction Contracts

### Check-In Form
- All fields are optional except `check_date` (auto-populated client-side as today's local ISO date)
- Validation: weight must be 20–300 kg if provided; hunger/stress 1–5; sleep 0–24h; activity 0–1440 min
- Inline error: `text-sm text-error bg-error/5 rounded-xl px-4 py-3` (matches `MealAnalysisForm.tsx` error pattern exactly)
- Submit button disabled while loading; shows `checkInSubmitting` copy during pending request
- On success: form hides, summary refreshes in-place (no full page reload), success message shown for 3 seconds then fades
- Re-submit same day: updates existing record (upsert) — no error shown to user, just success message

### Sparkline
- Renders only when `weights.length >= 2`
- RTL/LTR: sparkline reads left-to-right regardless of locale (time is always progressive left → right)
- No interaction on sparkline itself (no hover tooltips in v1) — `aria-hidden="true"`

### Behavior Win Chips
- Chips are read-only display — no tap action
- Achieved chip shows a checkmark (`✓` in `text-brand`) before the label
- Not-yet-tracked chips (protein, fiber, water for v1) show with `text-ink-3` and a `—` prefix

### Weekly Tab
- Weekly report generates on first load when `GET /api/v1/progress/weekly-report` is called
- Loading state: centered spinner (`w-8 h-8 rounded-full border-2 border-brand border-t-transparent animate-spin`)
- Empty state when fewer than 3 check-ins: shows `emptyWeeklyTitle` + `emptyWeeklyDesc` + link to check-in form

### Empty State (no check-ins at all)
- Renders above the tab bar when `has_data === false`
- Content: large soft illustration placeholder (120×120px `rounded-full bg-brand-muted` with a `Scale` icon at 48px `text-brand`), heading at 20px bold, body at 14px `text-ink-2 leading-relaxed`, CTA button matching the primary button style
- Tab bar hidden in empty state — show only after first check-in is submitted

## Accessibility Contract

- All form inputs have associated `<label>` elements (explicit `htmlFor` / `id` pairing)
- 1–5 scale buttons use `aria-pressed` or `aria-selected`; group has `role="group"` with `aria-label`
- Sparkline SVG: `aria-hidden="true"` — numeric weight delta text is the accessible equivalent
- Behavior win chips: `role="status"` with `aria-label` combining win name and achieved/not-achieved state
- Tab bar: `role="tablist"` + `role="tab"` + `aria-selected` on each tab
- Loading spinner: `role="status"` + `aria-label` from `common.loading`
- Empty state: heading at `h2` level, not a paragraph; CTA is a real `<button>` or `<Link>`
- Touch targets: all interactive elements minimum 44px height

## RTL/LTR Contract

- All layout uses Tailwind v4 logical properties exclusively: `ps-`, `pe-`, `ms-`, `me-`, `rounded-es-`, `rounded-ee-` — never `pl-`, `pr-`, `ml-`, `mr-`
- Sparkline always renders left-to-right (time progression) regardless of locale — this is intentional and correct
- `ChevronLeft` back button uses `getIconFlipClass(locale)` (existing utility from `@/lib/direction`) to flip in RTL
- Behavior win chip row is `flex-wrap` — wraps naturally in both directions
- Weekly metric rows: label on inline-start, value on inline-end (`justify-between`)
- Tab bar active indicator: `border-b-2 border-brand` — logical bottom border, direction-neutral
